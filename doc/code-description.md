# 程序代码描述

## 代码结构

程序主要代码结构如下：

```
.
├── src/
    ├── channels.c
    ├── ephemeris.c
    ├── errors.c
    ├── io.c
    ├── options.c
    ├── signal.c
    ├── siren.h
    ├── toml.c
    ├── toml.h
    └── utils.c
├── tools/
    ├── genpath.py
    ├── plotpath.py
    ├── plotskypath.py
    └── samebin.py
└── siren.c
```

### C 程序

包含在 `src` 文件夹内的所有文件以及 `siren.c` 采用 C 语言编写。其中，`ephemeris.c` 包含与广播星历相关的函数，如计算卫星位置和可见性；`errors.c` 为误差仿真相关，如电离层、对流层延迟误差等；`io.c` 与输入输出有关，例如广播星历文件、动态轨迹文件和调试信息的输入输出等；`options.c` 处理配置文件，如配置项默认值和配置文件的读取；`signal.c` 处理仿真信号，例如伪随机码生成、导航消息生成、伪距和载波相位计算和数据码相位计算等；`channels.c` 包含对数据通道的处理（信号生成和接收时，每颗卫星的处理可使用一组并行的软/硬件实现，称为通道）；`utils.c` 为一些基础的公用函数，例如坐标和时间系统的转换；`toml.h` 和 `toml.c` 用于解析 TOML 格式，这两个文件来自于 [TOML C library](https://github.com/cktan/tomlc99)，将其集成到代码里便于程序的跨平台编译。`siren.c` 是 CLI 程序的主程序，包含命令行参数解析和主程序入口。

`siren.h` 文件定义了生成 GPS 信号所需的结构体、常量和函数声明等内容。主要数据结构如下表：

| 结构名     | 含义                                                         |
| ---------- | ------------------------------------------------------------ |
| gtime_t    | 仿真系统采用的时间，以 GPS 周和周内秒表示                        |
| datetime_t | UTC 时，以年月日时分秒表示                                   |
| ephem_t    | 每颗 GPS 卫星的广播星历，对应与导航文件中的各项参数          |
| ionoutc_t  | 电离层延迟、跳秒等广播星历头文件中的信息                     |
| range_t    | 卫地距，包含时间、伪距、变化率、真实距离、卫星方位/高度角和电离层延迟 |
| channel_t  | 信号通道，包含卫星 PRN、码相位、载波相位、接收时间、卫地距等 |
| options_t  | 项目配置，包含信号、误差和输出格式等                         |

主要函数如下表：

| 函数名       | 输入                       | 输出                 | 作用                                     |
| ------------ | -------------------------- | -------------------- | ---------------------------------------- |
| gencode      | 卫星 PRN                   | C/A 码序列           | 对特定的卫星产生 C/A 码序列              |
| ltcmat       | 大地坐标                   | 矩阵（3×3）          | 计算大地坐标系到地心坐标系的过渡矩阵     |
| neu2azel     | 站心坐标                   | 方位角/高度角        | 站心坐标（NEU）转方位角/高度角           |
| satpos       | 卫星星历、GPS 时           | 卫星坐标、速度、钟差 | 由广播星历计算卫星坐标、速度和钟差       |
| eph2sbf      | 卫星星历、电离层           | 卫星子帧             | 将广播星历和电离层参数转为卫星子帧       |
| checksum     | 子帧数据                   | 校验码               | 计算卫星子帧数据的校验码                 |
| readnav      | 广播星历文件               | 星历、电离层数据     | 读取广播星历文件，将数据加载到程序       |
| ionodelay    | 电离层参数、位置时间       | 电离层延迟           | 根据电离层参数计算电离层延迟             |
| calcrange    | 星历、时间、接收机位置等   | 卫地距               | 根据星历、时间、接收机位置等计算卫地距   |
| calccaphase  | 信号通道、卫地距、时间差   | 信号通道             | 根据卫地距和时间差计算码相位             |
| readtraj_csv | 文件名                     | 坐标序列             | 从文件中读取用户位置序列                 |
| readtraj_gga | 文件名                     | 坐标序列             | 从 GPGGA 文件中读取用户位置序列          |
| gennavmsg    | 时间、信号通道、初始化标志 | 无                   | 生成导航消息数据                         |
| satvisible   | 星历、接收机位置、时间等   | 可见状态             | 检查卫星是否可见（高度角大于设定值）     |
| initchan     | 无                         | 无                   | 初始化通道，重设通道状态                 |
| allocchan    | 星历、接收机位置、时间等   | 已分配通道的个数     | 为接收机分配通道并返回可见卫星数         |
| updatechan   | 星历、接收机位置、时间等   | 更新通道的个数       | 在新的时刻和位置更新通道（的伪距和相位） |
| flushchan    | 星历、时间等               | 无                   | 刷新通道的星历                           |
| gensignal    | 缓冲区、缓冲区大小和间隔   | 信号I/O序列          | 生成一段时间内（0.1s）的I/Q信号          |
| writeiq      | 缓冲区、缓冲区大小和格式等 | 成功或失败状态       | 根据格式控制将缓冲区内I/Q信号写入文件    |

### Python 程序

文件夹 `tools` 中的程序采用 Python 语言编写，用于程序测试、输入输出数据准备和可视化等辅助功能。主要功能为：

| 程序名        | 功能                                               |
| ------------- | -------------------------------------------------- |
| genpath.py    | 根据路径转折点内插生成轨迹数据，作为动态模式的输入 |
| plotpath.py   | 轨迹可视化（需调用海外地图 API 作为底图）          |
| plotskymap.py | 卫星天空图可视化，显示仿真的卫星                   |
| samebin.py    | 判断两个二进制文件的内容是否一致，用于程序测试     |

各 Python 的使用方法可通过 `-h` 参数查看，不再赘述。

## 程序编译

推荐使用 GCC 编译程序（目前仅在 GCC 中进行过有效测试）。Windows 系统使用 GCC 需要下载和配置 [MinGW-w64](https://www.mingw-w64.org/)，配置完成后即可进行编译。Windows 系统进入 `siren` 文件夹后运行命令：

```
$ mingw32-make
```

GNU/Linux 系统进入 `siren` 文件夹后运行命令：

```
$ make
```

即可启动编译，编译完成后将看到可执行的程序文件。
