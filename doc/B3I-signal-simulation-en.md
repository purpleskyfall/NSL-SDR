# B3I signal simulation

The BeiDou B3I signal is one of the core publicly available signals of the BeiDou Satellite Navigation System (BDS), and its development has been closely linked to the BDS’s “three-step” strategic plan. During the BeiDou-2 phase, the B3I signal first provided RNSS services in the Asia-Pacific region, employing BPSK modulation at a code rate of 10.23 Mcps and supporting navigation message transmission at 50 bps. In the BeiDou-3 phase, the coverage of the B3I signal expanded globally, with positioning accuracy improved by 30% and availability enhanced by 5%, while also becoming compatible with satellite-based augmentation services. The B3I Interface Control Document (ICD) released in 2020 standardizes the signal parameters and supports high-precision positioning with dynamic decimeter-level and static centimeter-level accuracy.

## Signal Structure

The physical layer parameters of B3I are shown in the table below.

|      constants      |    Values    |
| :-----------------: | :----------: |
|  Carrier frequency  | 1268.520 MHz |
|  Ranging code rate  |  10.23 Mcps  |
| Ranging code length |    10230     |
|  Signal bandwidth   |  20.46 MHz   |

The structure of the simulated signal can be expressed as:
$$
\begin{equation}
S^j = P_c G^j(t) [c^j(t) N^j(t) D^j(t)]cos \phi^j + P_c G^j(t) [c^j(t) N^j(t) D^j(t)] sin \phi^j
\end{equation}
$$
In this equation, the superscript $j$ denotes the satellite number; $P_c$ represents the signal power; $G^{j}(t)$ is the signal gain at the receiver; $c(t)$ denotes the signal ranging code level at time $t$; $N(t)$ represents the NH code level of the secondary coding; and $D(t)$ denotes the data code level at the same time instant. The signal gain depends both on the propagation distance of the signal and on the elevation angle of the satellite, and thus it varies with time. We denote the first term on the right-hand side of the equation as $I_j(t)$ (referred to as the in-phase component) and the second term as $Q_j(t)$ (referred to as the quadrature component).

### Pseudorandom code

The pseudorandom code of the B3I signal is generated by two shift registers, and the generation formulas for the two sequences are as follows:
$$
\begin{equation}
G1(x) = x^{13}+x^{4}+x^{3}+x+1
\end{equation}
$$

$$
\begin{equation}
G2(x) = x^{13}+x^{12}+x^{10}+x^{9}+x^{7}+x^{6}+x^{5}+x+1
\end{equation}
$$

Among these, the G1 sequence contains 8,190 chips per period, with an initial phase of all 1s; the G2 sequence contains 8,191 chips per period, and its initial phase is determined by the PRN number. After generating the Gold code through modulo-2 addition, the sequence is truncated to 10,230 chips. The phase assignment for the G2 sequence can be referenced in the B3I Interface Control Document (ICD).

### Navigation message

Depending on their rate and structure, navigation messages are divided into D1 navigation messages and D2 navigation messages. The D1 navigation message has a rate of 50 bps and is modulated with a secondary code at a rate of 1 kbps. Its content includes basic navigation information (basic navigation data for this satellite, ephemeris data for all satellites, and time synchronization information with other systems). The D2 navigation message has a rate of 500 bps and contains both basic navigation information and wide-area differential information (differential and integrity information for the BeiDou system, as well as grid-point ionospheric information).

The B3I signal transmitted by MEO/IGSO satellites uses the D1 navigation message, while the B3I signal transmitted by GEO satellites uses the D2 navigation message. This program simulates only MEO/IGSO satellites; therefore, it is sufficient to generate only the D1 navigation message. For detailed information on the navigation message, refer to the B3I Interface Control Document (ICD). In fact, the structure of the B3I navigation message is completely identical to that of the B1I.

### Error-correcting coding

The navigation message data code is grouped in sequences of 11 bits each, and a 4-bit error-correction code is generated for each group. These 11-bit information segments, together with their corresponding 4-bit error-correction codes, form a BCH code that is 15 bits long. Every two groups of BCH codes undergo a parallel-to-serial conversion, bit by bit, to produce an interleaved code that is 30 bits long. In other words, each 30-bit interleaved code contains 22 bits of data code and 8 bits of error-correction code. In fact, the error-correction coding scheme used for the B3I navigation message is entirely identical to that used for the B1I navigation message.

### Secondary encoding

The secondary modulation of the D1 navigation message refers to modulating a Neumann-Hoffman code (referred to as the NH code) onto the D1 navigation message at a rate of 50 bps. This NH code has a period equal to the width of one navigation information bit (20 ms). The width of one bit in the NH code is the same as that of the ranging code (both 1 ms), and the code rate is 1 kbps. The NH code is modulated synchronously with the ranging code and the navigation information code using modulo-2 addition. The 20-bit NH code is: 00000100110101001110. In practice, the secondary modulation method for the B3I navigation message is entirely identical to that used for the B1I message.

## Error model

Consistent with B1I, the BeiDou B3I navigation message provides an 8-parameter Klobuchar model, which can be used to model ionospheric delay errors. For process delay errors, the Saastamoinen model is employed for approximate calculation.
