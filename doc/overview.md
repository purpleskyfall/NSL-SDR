## 背景知识

### 什么是 SDR

软件无线电（Software Defined Radio，SDR）是一种无线电通信技术，它使用软件来实现无线电信号的调制解调和信号处理等功能。与传统的无线电通信系统相比，SDR系统具有更高的灵活性、可编程性和适应性，可以用于多种不同的应用场景。

SDR 系统的主要特点是使用数字信号处理技术来完成无线电信号的调制解调和信号处理等功能。SDR 系统由硬件和软件两部分组成。硬件部分包括调制解调器、天线、射频前端等组件，用于将无线电信号转换为数字信号（或将数字信号转换为无线电信号）。软件部分包括处理器、编程算法等组件，用于对数字信号进行处理、调制解调和信号处理等操作。通过软件的控制，SDR 系统可以灵活地改变调制方式、频率、带宽等参数，以适应不同的应用需求。

SDR 技术广泛应用于多种无线电通信系统，如广播电视、卫星通信、移动通信、军事通信等。该技术可以提高通信系统的灵活性和可编程性，降低系统成本和功耗，同时也可以提高系统的安全性和抗干扰性能。

常见的 SDR 硬件包括：

- [通用软件无线电外设（USRP）](http://www.ettus.com/)：USRP 是由 Ettus Research 公司开发的一种 SDR 硬件平台，它可以通过插件式的射频前端和数字信号处理板卡实现不同的无线电通信系统。USRP 具有高灵活性、高可扩展性和高性能等优点，广泛应用于无线电通信、雷达、信号处理等领域。

- [BladeRF](https://www.nuand.com/bladerf-1/)：BladeRF 是由 Nuand 公司开发的一款 SDR 硬件平台。它采用了 FPGA 和 ARM Cortex-A9 处理器，可以支持多种无线电通信标准和频率范围。BladeRF 具有高性能、低功耗和可编程性等特点。

- [HackRF](https://greatscottgadgets.com/hackrf/one/)：HackRF 是一款 SDR 硬件平台，由 Great Scott Gadgets 公司开发。它可以实现从 1MHz 到 6GHz 的频率范围内的无线电接收和发射，支持多种调制方式和编码格式。HackRF 具有高性能、低成本和开放源代码等优点。

### SDR 的文件格式

IQ 文件是 SDR 技术常用的无线电信号文件格式，它记录无线电信号的 I（实数）和 Q（虚数）分量，可以用于离线信号处理和分析。IQ 文件通常使用标准的二进制格式，可以被大多数 SDR 软件和工具所支持。软件无线电的软件平台将无线电信号以很高的采样率进行采样，然后将每个采样点的 I/Q 信号以数字编码，就生成了 IQ 数据信号文件。

IQ 文件在存储无线电信号时，一般用单字节（8 比特）或双字节（16 比特）的整数来表示信号的振幅。显然，用双字节的整数描述信号更“精细”，但同时也将占用更大的存储空间。

### GNSS 信号的组成

以 GPS 为例，GPS（Global Positioning System）信号主要由以下三个组成部分构成：

1. 导航消息（Navigation Message）：导航消息是 GPS 信号的基础信息，包含了卫星的星历、时间、健康状况等信息。导航消息以 50 bps（比特/秒）的码流传输，每个码流包含了多个导航数据段和纠错码等信息。
2. 载波波形（Carrier Waveform）：载波波形是 GPS 信号的高频信号部分，用于传输导航消息和定位信息。GPS 信号的载波波形频率为 L1 频段（1575.42 MHz）和 L2 频段（1227.60 MHz），是一种相位连续的正弦波信号。
3. 伪随机噪声码（Pseudo-Random Noise Code）：伪随机噪声码是 GPS 信号的伪码部分，用于区分不同的卫星信号和抵御干扰。伪随机噪声码是一种由二进制序列组成的伪码序列，长度为 1023 个码片。

在 GPS 信号的传输过程中，伪随机噪声码和导航消息被调制到载波波形上进行传输。接收机根据解调得到的伪随机噪声码和导航消息，可以计算出卫星和接收机之间的距离和卫星坐标，从而进一步解算得到自身的位置信息。

### GNSS 接收机对信号的处理

GNSS（Global Navigation Satellite System）接收机处理收到的信号的过程包括以下几个步骤：

1. 信号接收：接收机通过天线接收信号，然后将信号转换为电信号传输到接收机的前置放大器中。
2. 信号放大：接收机的前置放大器将接收到的信号进行放大，增强信号强度，以便后续处理。
3. 信号滤波：信号在传输过程中会受到多路径效应、噪声干扰等影响，需要对信号进行滤波，去除噪声和干扰，保留所需的信号成分。
4. 信号混频：接收机将载波波形和伪随机噪声码进行混频，得到中频信号，使信号频率降低到容易处理的范围内。
5. 信号解调：接收机将混频后的信号进行解调，得到导航消息和伪随机噪声码等信息。
6. 信号比对：接收机将接收到的伪随机噪声码和导航消息与已知的卫星信息比对，确定卫星的编号和时间信息等。
7. 信号处理：接收机持续根据信号的伪随机噪声码和相位，计算出卫星和接收机之间的距离和位置等信息，完成定位和导航功能。

需要注意的是，GNSS 接收机的性能受多种因素影响，如信号强度、天线位置、多路径效应、卫星分布等。为了提高 GNSS 接收机性能，需要采用合适的天线和前置放大器、优化信号滤波和解调算法等措施。

## Siren

Siren 是基于软件无线电技术的 GPS 信号仿真软件，能够模拟 GPS 卫星发射的信号，并生成相应的 IQ 信号文件，用于测试 GPS 接收机的性能。该项目主要基于 C 语言开发，通过简单的配置（主要是卫星星历和用户位置），即可生成对应的 GPS 仿真信号 IQ 文件，将仿真信号以设定的频率发射，周边的接收设备即可接收到仿真的信号。

### 程序原理

GPS 信号仿真是一种通过模拟 GPS 卫星发出的信号，来测试和评估 GPS 接收机性能的方法。其原理可以简单概括为以下几个步骤：

1. 生成卫星轨道和时钟数据：仿真系统需要生成卫星的轨道数据和时钟数据，以模拟 GPS 卫星发射的信号。这些数据可以从真实的 GPS 卫星中获取，或者使用模型进行生成。
2. 生成信号：仿真系统根据卫星的轨道和时钟数据，生成 GPS 信号。GPS 信号是通过将多个频段的载波信号调制成一个复合信号来实现的。仿真系统需要模拟这个过程，以生成与实际相似的信号。
3. 传输信号：仿真系统将生成的 GPS 信号通过天线传输到接收机，以模拟真实的 GPS 信号传输过程。在传输过程中，信号可能会受到多路径效应、衰减和干扰等影响。
4. 接收信号：GPS 接收机接收仿真信号，并计算自身的位置和时间。
5. 评估性能：使用 GPS 接收机输出的位置和时间来评估其性能。例如，可以比较仿真系统计算出的位置和时间与真实值之间的误差，评估接收机的定位精度和时钟同步精度。

Siren 程序主要工作在上述步骤的 1-2。传输信号由相关的 SDR 硬件如 [HackTF]([HackRF One - Great Scott Gadgets](https://greatscottgadgets.com/hackrf/one/))，[BladeRF]([bladeRF - Nuand](https://www.nuand.com/bladerf-1/)) 和 [USRP](http://www.ettus.com/) 等完成。Siren 软件的工作包括以下几个步骤：

1. 生成卫星轨道：首先需要生成模拟的 GPS 卫星轨道，软件使用 RINEX 格式的广播星历文件生成卫星轨道数据，卫星轨道数据通常包含卫星的位置、速度、卫星钟差等信息，用于计算信号的传播路径和时间延迟等参数。
2. 生成导航消息：导航消息是 GPS 信号中的重要组成部分，包含了卫星的星历、时间、卫星健康状况等信息。
3. 生成伪随机噪声码：伪随机噪声码是 GPS 信号中的另一个重要组成部分，用于区分不同的卫星信号和抵御干扰。
4. 生成载波波形：GPS 信号的载波包含 L1 频段（1575.42 MHz）和 L2 频段（1227.60 MHz），是一种相位连续的正弦波信号，目前程序仅支持 L1 频段。
5. 信号调制：信号调制是将导航消息和伪随机噪声码调制到载波上，得到每个采样点的 I/Q 信号值，并对所有可见卫星累加得到最终信号。
6. 信号模拟输出：将调制后的信号输出到文件中，输出信号的格式是二进制的 IQ 文件，采样点的数值宽度可以是 1 比特、8 比特或 16 比特。

### 主程序流程

忽略必要的配置参数加载、配置项检查等过程，主程序流程如下图：

```flow
start=>start: 开始
end=>end: 结束
eph=>operation: 星历加载与选择
visible=>operation: 查找可见卫星
rho=>operation: 计算卫地距
range=>operation: 计算C/A码和数据码相位
iq=>operation: 生成信号
cycle=>condition: 更新星历？
endt=>condition: 达到仿真时长
start->eph->visible->rho->range->iq->endt
endt(no)->cycle
endt(yes)->end
cycle(yes)->eph
cycle(no)->rho
```

注：程序在每分钟的第 0 秒和第 30 秒检查是否需要更新星历。
